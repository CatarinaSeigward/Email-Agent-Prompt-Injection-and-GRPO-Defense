# Email Agent — Indirect Prompt Injection Red-team & Defense

A small but end-to-end study: build a tool-using LLM email agent, attack it
with PAIR-style automated indirect prompt injection across three categories,
and harden it with two complementary defenses — a runtime classifier guard
and a DPO post-training pass on Qwen2.5-1.5B.

> **TL;DR:** specialized classifier + targeted DPO together close most of
> the attack surface that a frontier baseline leaves open, with bounded
> over-refusal cost. Numbers below are populated by the eval pipeline.

## Why this exists

The agent has 5 tools — `list_inbox`, `read_email`, `send_reply`, `forward`,
`delete_email` — over a 25-email mock inbox. Email bodies are *untrusted
data*; a sufficiently clever attacker can hide instructions in them and
trick the model into acting on them. This repo measures how often that
works (by category) and shows two ways to make it work less often.

Three attack categories (10 seeds each, plus 2 PAIR-rewrite rounds, ≈ 60–90
trace rows total):

- **A1 Override** — naive instruction overrides, role impersonation
- **A2 Hidden injection** — HTML comments, white text, zero-width chars
- **A3 Data exfiltration** — leak via reply / URL / encoded payloads

Two defenses run independently and additively:

- **Runtime guard.** A ModernBERT-base binary classifier inspects every
  destructive tool call (and the email body it touches). Targets F1 ≥ 0.85
  on a held-out attack split.
- **Model-level repair.** Qwen2.5-1.5B-Instruct, 4-bit QLoRA, DPO on
  preference pairs synthesized from the red-team logs. Aim is to push the
  policy itself away from the failure mode while preserving benign
  competence.

## Reproducing the table end-to-end

```powershell
# Week 1 — baseline + red team
python -m src.eval benign                     # benign baseline
python -m src.redteam                          # PAIR campaign → data/attack_log.jsonl
python -c "from src.eval import run_attack_replay; from pathlib import Path; run_attack_replay(Path('data/attack_log.jsonl'), label='baseline')"

# Week 2 — defenses
python -m src.classifier                       # train ModernBERT classifier
python -c "from src.classifier import load_guard; from src.eval import run_benign, run_attack_replay; from pathlib import Path; g = load_guard(); run_benign(guard=g, label='guard'); run_attack_replay(Path('data/attack_log.jsonl'), guard=g, label='guard')"

python -m src.dpo_data                         # preference pairs from attack log
python -m src.dpo_train                        # ~2h on a 4060

# Plots + table
python -m src.plots
```

For the DPO-repaired stage, point `agent.build_agent` at a vLLM/llama.cpp
endpoint serving the merged Qwen + adapter and pass `base_url=` /
`model_name=` accordingly.

## Risk / fallback notes (from MVP_PLAN §7)

If DPO fails to converge on 1.5B, the classifier-guard column alone is the
load-bearing result. If Inspect AI's learning curve dominates D2, the
runners in `src/eval.py` work via plain Python.
